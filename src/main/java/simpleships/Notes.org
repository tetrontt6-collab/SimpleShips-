* Notes
** Connecting Persistent Properties
   All ship components will have a property for the key
   Constants.ITEM_TYPE_KEY in the component's PersistentData settings.
   
*** Seats
    The armor stand (helmAnchor) has these properties
      - Constants.ITEM_TYPE_KEY = Constants.SHIP_HELM_ITEM_TYPE
      - Constants.SHIP_HELM_ID_KEY = <the Ship's unique id str>

    The block display for the post has these properties
      - Constants.ITEM_TYPE_KEY = Constants.SHIP_HELM_POST_TYPE
      - Constants.SHIP_HELM_ID_KEY = <the Ship's unique id str>

    The block display for the seat has these properties
      - Constants.ITEM_TYPE_KEY = Constants.SHIP_HELM_SEAT_TYPE
      - Constants.SHIP_HELM_ID_KEY = <the Ship's unique id str>

*** EntityPad
    Each EntityPad gets a unique UUID on construction.
    
    The armor stand (post) has these properties
    - Constants.ITEM_TYPE_KEY = Constants.ENTITY_PAD_ITEM_TYPE
    - Constants.ENTITY_PAD_ID_KEY = uuid

    The block display nest has these properties
    - Constants.ITEM_TYPE_KEY = Constants.ENTITY_PAD_ITEM_TYPE
    - Constants.ENTITY_PAD_ID_KEY = uuid

    The interaction has these properties
    - Constants.ITEM_TYPE_KEY = Constants.ENTITY_PAD_ITEM_TYPE
    - Constants.ENTITY_PAD_ID_KEY = uuid

*** PassengerSeat
    Each PassengerSeat gets a unique UUID on construction

    The ArmorStand (stand) has these properties
    - Constants.ITEM_TYPE_KEY = Constants.PASSENGER_SEAT_ITEM_TYPE
    - Constants.PASSENGER_SEAT_ID_KEY = uuid

    The BlockDisplay (stair) has these properties
    - Constants.ITEM_TYPE_KEY = Constants.PASSENGER_SEAT_ITEM_TYPE
    - Constants.PASSENGER_SEAT_ID_KEY = uuid

    The BlockDisplay (cushion) has these properties 
    - Constants.ITEM_TYPE_KEY = Constants.PASSENGER_SEAT_ITEM_TYPE
    - Constants.PASSENGER_SEAT_ID_KEY = uuid
